## 一、概述

### 1.1 项目背景

即梦AI是字节跳动旗下的AI图片生成服务，通过火山引擎开放平台提供API接口。本项目旨在开发一个基于Python后端的Web应用程序，封装即梦AI图片生成API，为用户提供便捷的Web界面，支持文生图、图生图及局部重绘等AI图片生成与编辑能力。

### 1.2 参考文档

| 文档名称 | 说明 | 链接 |
|---------|------|------|
| 即梦AI-图片生成4.6-接口文档 | 火山引擎即梦AI图片生成API接口规范 | https://www.volcengine.com/docs/85621/2275082?lang=zh |

### 1.3 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 使用Web界面进行AI图片生成和编辑的用户。单用户系统，不涉及多角色权限管理。 |

## 二、用户故事

### US-001 - 配置API密钥

作为用户，我希望通过环境变量配置文件（.env）来管理火山引擎的AccessKey和SecretKey，以便程序启动时自动完成鉴权初始化，无需每次使用时重复输入。

**验收场景**：
1. **Given** 程序启动，**When** 读取.env文件中的VOLC_ACCESS_KEY和VOLC_SECRET_KEY环境变量，**Then** 成功初始化API客户端，程序正常运行。
2. **Given** .env文件中缺少密钥信息，**When** 程序启动检测，**Then** 提示用户配置密钥并退出。

### US-002 - 文生图（文本生成图片）

作为用户，我希望在Web页面中输入文本描述（提示词），系统调用即梦AI的文生图API生成图片，并保存到本地，以便快速将创意想法转化为视觉图像。

**验收场景**：
1. **Given** 用户进入文生图页面，**When** 输入正向提示词、选择图片尺寸、图片数量、风格、调节CFG Scale和Seed值，点击"生成"按钮，**Then** 系统提交任务并异步等待生成完成，完成后显示生成的图片预览并保存到本地。
2. **Given** 文生图页面，**When** 用户输入负面提示词，**Then** 系统将此参数传入API，确保生成结果规避不希望出现的内容。
3. **Given** 文生图任务提交后，**When** API返回任务ID，**Then** 系统自动轮询查询任务状态，并在前端展示处理中的进度状态。

### US-003 - 图生图（以图生图）

作为用户，我希望上传一张参考图片并配合文本描述，调用即梦AI的图生图API生成与参考图片风格或内容相关的新图片。

**验收场景**：
1. **Given** 用户进入图生图页面，**When** 上传本地图片文件、输入提示词、选择尺寸和风格等参数，点击"生成"，**Then** 系统上传图片并调用图生图API，生成结果后展示并保存。
2. **Given** 图生图页面，**When** 用户未上传参考图片就点击生成，**Then** 系统提示"请先上传参考图片"。

### US-004 - 局部重绘

作为用户，我希望上传一张图片并在网页画布中用画笔标记要修改的区域，配合文本描述进行局部重绘，实现"指哪改哪"的图片编辑效果。

**验收场景**：
1. **Given** 用户进入局部重绘页面，**When** 上传图片、用画布上的画笔工具标记重绘区域、输入新的描述提示词，点击"生成"，**Then** 系统将原图和遮罩图传入即梦AI的局部重绘API，生成修改后的图片。
2. **Given** 局部重绘画布，**When** 用户使用橡皮擦工具擦除已标记的区域，**Then** 擦除部分恢复为未选中状态。
3. **Given** 局部重绘页面，**When** 用户调整画笔大小，**Then** 画布上的画笔粗细随之变化。

### US-005 - 历史生成记录管理

作为用户，我希望在Web页面上查看历史图片生成记录，包括每次生成的提示词、生成时间、生成参数和图片缩略图，以便回顾和管理之前生成的内容。

**验收场景**：
1. **Given** 用户进入历史记录页面，**When** 页面加载，**Then** 展示历史生成记录列表，包含提示词文本、生成时间、图片缩略图。
2. **Given** 历史记录列表，**When** 用户点击某条记录，**Then** 展示该次生成的详细信息（参数、生成结果大图）。
3. **Given** 历史记录列表，**When** 用户点击删除按钮，**Then** 删除该条记录及其本地图片文件。

### 边界情况

- 当API调用失败（网络异常、鉴权失败、配额超限等）时，系统应在前端展示明确错误信息，不导致程序崩溃。
- 当图片生成任务超时（超过预设时间仍未完成），系统应提示用户任务超时并允许重新提交。
- 当上传的图片格式不支持或文件过大时，系统应给出格式/大小限制提示。
- 当用户重复提交相同参数的生成请求时，系统应正常处理（不强制去重，但允许用户取消正在执行的任务）。

## 三、功能需求

### 3.1 API密钥管理

- **FR-001 [US-001]**：系统必须从.env环境变量文件中读取VOLC_ACCESS_KEY和VOLC_SECRET_KEY配置，用于即梦AI API鉴权初始化。
- **FR-002 [US-001]**：系统启动时须校验API密钥配置是否完整，缺失时给出明确提示并阻止非鉴权的API调用。
- **FR-003 [US-001]**：系统必须实现火山引擎AK/SK签名认证（HMAC-SHA256），所有API请求携带有效的鉴权头。

### 3.2 文生图功能

- **FR-004 [US-002]**：系统必须提供Web页面，支持用户输入正向提示词（支持中英文）作为文生图的文本描述。
- **FR-005 [US-002]**：系统必须支持用户输入负面提示词，用于排除不希望在图片中出现的内容。
- **FR-006 [US-002]**：系统必须提供图片尺寸选择功能，支持以下预设尺寸：1:1方图(1024×1024)、16:9横图(1920×1080)、9:16竖图(1080×1920)、4:3横图，以及用户自定义宽高输入。
- **FR-007 [US-002]**：系统必须支持用户选择生成图片数量（1~4张）。
- **FR-008 [US-002]**：系统必须提供预设风格选项供用户选择（如写实摄影、动漫、水彩、油画、3D渲染等），对应即梦AI API的风格参数。
- **FR-009 [US-002]**：系统必须支持用户调节CFG Scale（提示词相关性）参数，提供滑块或数值输入。
- **FR-010 [US-002]**：系统必须支持用户指定随机种子（Seed）值，默认值为0（随机），用户可手动输入指定值用于复现相同生成效果。
- **FR-011 [US-002]**：系统提交文生图任务后，须异步轮询即梦AI API的生成结果，并在前端展示任务处理中的加载状态。
- **FR-012 [US-002]**：系统生成完成后，须在前端展示生成结果图片，并将图片自动保存到本地指定目录。

### 3.3 图生图功能

- **FR-013 [US-003]**：系统必须提供Web页面，支持用户上传本地的参考图片文件（支持JPG、PNG格式，单文件不超过20MB）。
- **FR-014 [US-003]**：系统必须将上传的参考图片转换为即梦AI API所需的格式（Base64编码或上传URL），连同文本提示词一并提交图生图API。
- **FR-015 [US-003]**：系统必须在前端校验：未上传图片时点击生成按钮应给出"请先上传参考图片"的提示。
- **FR-016 [US-003]**：图生图页面同样支持FR-004~FR-012中描述的正向提示词、负面提示词、尺寸、数量、风格、CFG Scale、Seed等参数。

### 3.4 局部重绘功能

- **FR-017 [US-004]**：系统必须提供前端画布交互组件，用户上传图片后可在画布上使用画笔工具标记需要重绘的区域。
- **FR-018 [US-004]**：画布工具必须支持画笔大小调节和橡皮擦擦除功能。
- **FR-019 [US-004]**：系统必须将用户标记的重绘区域生成黑白遮罩图（Mask），白色区域为需要重绘的部分。
- **FR-020 [US-004]**：系统必须将原图和遮罩图一并提交至即梦AI局部重绘API，配合用户输入的文本提示词进行局部修改。
- **FR-021 [US-004]**：局部重绘页面同样支持FR-004~FR-012中的提示词、尺寸等参数（尺寸与原图一致）。

### 3.5 历史记录管理

- **FR-022 [US-005]**：系统必须记录每次图片生成的历史信息，包括：提示词文本、生成时间、图片尺寸、生成数量、风格、Seed值、CFG Scale、生成的图片文件路径。
- **FR-023 [US-005]**：系统必须提供历史记录列表页面，按生成时间倒序展示，每条记录显示提示词摘要、生成时间和图片缩略图。
- **FR-024 [US-005]**：系统必须支持用户点击历史记录查看详情，包括完整参数和生成大图。
- **FR-025 [US-005]**：系统必须支持用户删除单条历史记录，删除时须同时移除对应的本地图片文件。

### 3.6 图片保存与展示

- **FR-026 [US-002][US-003][US-004]**：系统生成的所有图片须按日期和任务ID组织目录结构保存到本地，如 `./output/YYYYMMDD/{task_id}_1.png`。
- **FR-027 [US-002][US-003][US-004]**：系统在前端页面展示图片时须支持点击查看原图大图（或在新标签页打开）。
- **FR-028 [US-005]**：系统在历史记录中展示的图片缩略图须支持点击放大预览。

## 四、数据实体

### 4.1 核心数据实体

- **生成记录（GenerationRecord）**：记录单次图片生成请求的全量信息，是系统核心业务实体。
- **图片（GeneratedImage）**：记录单张生成图片的元数据，与生成记录为一对多关系。

**实体关系**：
- 生成记录（GenerationRecord）与图片（GeneratedImage）是一对多关系（一次生成请求可产出多张图片）。

### 4.2 核心字段规则

#### GenerationRecord（生成记录）

| 字段名 | 说明 | 是否必填 | 取值/规则 |
|--------|------|---------|----------|
| id | 记录唯一标识 | 是 | UUID，自动生成 |
| task_type | 任务类型 | 是 | 枚举：text2img / img2img / inpainting |
| prompt | 正向提示词 | 是 | 文本，支持中英文，最大1000字符 |
| negative_prompt | 负面提示词 | 否 | 文本，最大500字符 |
| image_size | 图片尺寸 | 是 | 格式"宽x高"，如"1024x1024" |
| style | 图片风格 | 否 | 字符串，对应预设风格名称 |
| cfg_scale | 提示词相关性 | 是 | 浮点数，范围1.0~20.0，默认7.0 |
| seed | 随机种子 | 是 | 整数，0表示随机，指定值用于复现 |
| image_count | 生成数量 | 是 | 整数，1~4 |
| reference_image_path | 参考图片路径 | 否 | 仅img2img和inpainting类型时非空 |
| mask_image_path | 遮罩图片路径 | 否 | 仅inpainting类型时非空 |
| api_task_id | 即梦AI任务ID | 是 | 提交API后返回的任务ID |
| status | 任务状态 | 是 | 枚举：pending / processing / completed / failed |
| error_message | 错误信息 | 否 | 任务失败时记录错误描述 |
| created_at | 创建时间 | 是 | ISO 8601时间戳，自动生成 |
| updated_at | 更新时间 | 是 | ISO 8601时间戳，状态变化时更新 |

#### GeneratedImage（生成图片）

| 字段名 | 说明 | 是否必填 | 取值/规则 |
|--------|------|---------|----------|
| id | 图片唯一标识 | 是 | UUID，自动生成 |
| generation_record_id | 所属生成记录ID | 是 | 关联GenerationRecord.id |
| image_index | 图片序号 | 是 | 整数，从1开始递增 |
| file_path | 本地文件路径 | 是 | 相对于output目录的路径 |
| file_size | 文件大小 | 否 | 整数，单位字节 |
| created_at | 创建时间 | 是 | ISO 8601时间戳，自动生成 |

## 六、项目约束与限制

- 即梦AI图片生成API为异步接口，调用后返回任务ID，需要通过轮询方式获取生成结果。
- 程序依赖火山引擎Python SDK（volcengine-python-sdk）进行API调用和鉴权。
- 用户需自行在火山引擎控制台开通即梦AI服务并获取AccessKey和SecretKey。
